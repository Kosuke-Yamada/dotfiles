# Doc Co-Authoring

ドキュメントの共同作成ワークフロー。

## 3つのステージ

### Stage 1: コンテキスト収集

背景情報を収集する：
- ドキュメントの種類は？
- ターゲットオーディエンスは？
- 望む影響・効果は？
- テンプレートや制約はあるか？
- 組織的・技術的コンテキストは？

**ユーザーに「持っているコンテキストをすべて出してください」と促す。**

### Stage 2: 精緻化と構造化

セクションごとに構築する反復サイクル：

1. **明確化の質問** - 不明点を確認
2. **ブレインストーミング** - 5-20のオプションを提案
3. **キュレーション** - ベストなものを選択
4. **ギャップチェック** - 不足を確認
5. **ドラフト作成** - コンテンツを執筆
6. **反復的改善** - フィードバックに基づき修正

**ヒント**: 全体を再印刷するのではなく、`str_replace`で外科的編集を行う。

### Stage 3: 読者テスト

新鮮な読者にドキュメントが機能するか検証：
- サブエージェントで読者の質問をシミュレート
- または「新しいClaude会話でテスト」を指示

**目的**: 盲点を発見する。

## 重要なポイント

- **ユーザーが主導権を持つ** - ステージをスキップしたり、ペースを調整可能
- **品質 > スピード**
- **最終セルフレビュー** - 完了前に確認
- **「このドキュメントはユーザーのもの」**
