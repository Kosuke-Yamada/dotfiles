# Frontend Design

独自性のある、プロダクショングレードのフロントエンドインターフェースを作成します。

## 設計原則

コーディング前に、明確な美的方向性を確立する：

- インターフェースの目的とオーディエンス
- 大胆なトーン選択（ミニマリスト、マキシマリスト、レトロ、オーガニック、ラグジュアリー、ブルータリストなど）
- 技術的制約
- 何がデザインを記憶に残るものにするか

## コアガイドライン

### タイポグラフィ
- 特徴的で予想外のフォントを選択
- 一般的な選択肢をデフォルトにしない

### カラー＆テーマ
- CSS変数を使用した一貫したパレット
- 支配的な色とシャープなアクセント

### モーション
- インパクトのあるアニメーション
- ページロード時のスタガードリビール

### 空間構成
- 非対称性、オーバーラップ、グリッドを壊す要素
- 独自性のあるレイアウト

### ビジュアルディテール
- テクスチャ、グラデーション、パターン
- コンテキスト固有のエフェクト

## 避けるべきこと

「汎用的なAI生成の美学」を避ける：
- 使い古されたフォントファミリー
- 陳腐なカラーグラデーション
- 予測可能なレイアウト
- 定型的なパターン

すべてのデザインは、意図的でコンテキスト固有の選択を反映すべき。
